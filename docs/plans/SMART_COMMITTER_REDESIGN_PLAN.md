## Общая концепция

Страница Smart Committer — не просто витрина, а **демонстрация работы системы в миниатюре**. Посетитель видит, как данные текут, как AI обрабатывает, как система маскируется. Всё интерактивно, но не перегружено.

## Этап 3: Интерактивная архитектура

### Концепция
SVG-схема системы с анимированным потоком данных между компонентами.

### Компоненты схемы
```
┌─────────────┐
│  Telegram   │ ◄── входящие сообщения
│   Monitor   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  AI Engine  │ ◄── обработка (смена цвета точек)
│ (GPT/Claude)│
└──────┬──────┘
       │
  ┌────┴────┐
  ▼         ▼
┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐
│Acct │  │ DB  │  │Anlt │  │ Sec │
│Mgr  │  │     │  │     │  │     │
└─────┘  └─────┘  └─────┘  └─────┘
```

### Поведение
1. **Idle:** Точки-данные постоянно «летают» между узлами
2. **Hover на узле:** 
   - Узел подсвечивается
   - Остальные затемняются (opacity: 0.3)
   - Связи с этим узлом ярче
3. **Click на узле:**
   - Появляется tooltip с деталями:
     - Название компонента
     - Технологии
     - Ключевые метрики

### Технические детали

**SVG структура:**
```svg
<svg class="arch-diagram">
  <!-- Узлы -->
  <g class="node" data-component="telegram">...</g>
  <g class="node" data-component="ai">...</g>
  ...
  
  <!-- Связи (линии) -->
  <path class="connection" d="..."/>
  
  <!-- Анимированные точки -->
  <circle class="data-particle">
    <animateMotion dur="2s" repeatCount="indefinite">
      <mpath href="#path-telegram-ai"/>
    </animateMotion>
  </circle>
</svg>
```

**JS:**
- Класс `ArchitectureDiagram`
- Обработка hover/click событий
- Управление состоянием (active node, tooltips)

### Файлы
- `smart-committer.html`: SVG-код схемы
- `smart-committer.css`: стили узлов, связей, анимаций
- `smart-committer.js`: класс `ArchitectureDiagram`

## Этап 4: Демо-чат с AI

### Концепция
Интерактивный мини-чат, демонстрирующий работу AI + TypingSimulator.

### UI
```
┌─────────────────────────────────────────┐
│  💬 Smart Committer Demo                │
├─────────────────────────────────────────┤
│                                         │
│  Выберите сообщение:                    │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │ 👤 "Привет, кто тут?"           │    │
│  └─────────────────────────────────┘    │
│  ┌─────────────────────────────────┐    │
│  │ 👤 "Что думаете о крипте?"      │    │
│  └─────────────────────────────────┘    │
│  ┌─────────────────────────────────┐    │
│  │ 👤 "Посоветуйте что почитать"   │    │
│  └─────────────────────────────────┘    │
│                                         │
│  ─────────────────────────────────────  │
│                                         │
│  🤖 [Печатает...]                       │
│                                         │
│  "Прив|"  ← курсор                      │
│                                         │
├─────────────────────────────────────────┤
│  Human Score: ████████████░░ 94%        │
│  Typing Pattern: Natural                │
└─────────────────────────────────────────┘
```

### Сценарии (5-7 штук)

| # | Входящее сообщение | Варианты ответа (рандом) |
|---|-------------------|--------------------------|
| 1 | "Привет, кто тут?" | "Прив!) тоже тут сижу, читаю" / "Привет, да вот залипаю в канале" |
| 2 | "Что думаете о крипте?" | "Ну смотря какой... битка норм держится" / "Сложная тема, я бы осторожно" |
| 3 | "Посоветуйте что почитать" | "Зависит от темы, тебе про что?" / "Недавно читал... [название], зашло" |
| 4 | "Кто-нибудь пробовал...?" | "Я пробовал, но давно уже" / "Не, не сталкивался пока" |
| 5 | "Админ тут?" | "Хз, вроде иногда появляется" / "Видел его недавно в чате" |

### Эффект TypingSimulator

1. Начало печати с задержкой 500-1500ms
2. Печать посимвольно:
   - Обычная скорость: 50-150ms между символами
   - Паузы на пробелах: +100-200ms
   - Паузы на знаках препинания: +200-400ms
3. Опечатки (10% шанс):
   - Напечатать неправильный символ
   - Пауза 300ms
   - Backspace
   - Правильный символ
4. «Задумывание» (5% шанс):
   - Пауза 500-1000ms посреди слова

### Метрики
- **Human Score:** Рассчитывается на основе:
  - Вариативность тайминга: +30%
  - Наличие опечаток: +20%
  - Естественные паузы: +25%
  - Длина ответа: +25%
- Показывается как progress bar

### Файлы
- `smart-committer.html`: разметка чата
- `smart-committer.css`: стили чата, сообщений, индикаторов
- `smart-committer.js`: классы `DemoChat`, `TypingSimulator`

## Этап 5: Антидетект + Наблюдатель

### Концепция
Визуализация работы системы антидетекта с интерактивным «глазом наблюдателя».

### UI
```
┌──────────────────────────────────────────────────────┐
│  🛡️ Система антидетекта                              │
│                                                      │
│  ┌────────────────────────────────────────────────┐  │
│  │                                                │  │
│  │     [Визуализация нейросети]                   │  │
│  │                                                │  │
│  │     ○ ─── ○ ─── ○                              │  │
│  │      \   / \   /                               │  │
│  │       ○ ─── ○           👁️ ← появляется        │  │
│  │      /   \ /   \            при вкл.           │  │
│  │     ○ ─── ○ ─── ○                              │  │
│  │                                                │  │
│  └────────────────────────────────────────────────┘  │
│                                                      │
│  ┌──────────────────┐                                │
│  │ 👁️ Наблюдатель   │  ○───────● (toggle)           │
│  └──────────────────┘  ВЫКЛ      ВКЛ                 │
│                                                      │
│  Detection Risk: ███░░░░░░░ LOW                      │
│                                                      │
│  Режим: Conservative                                 │
│  • Задержки между сообщениями: 2-5 мин              │
│  • Вариативность контента: Высокая                   │
│  • Ротация аккаунтов: Активна                        │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### Поведение

**Наблюдатель ВЫКЛ:**
- Нейросеть работает в обычном режиме
- Быстрые, ровные анимации
- Яркие связи между узлами

**Наблюдатель ВКЛ:**
1. Глаз появляется справа от сети
2. Глаз периодически «смотрит» на сеть (каждые 3-5 сек)
3. Когда глаз смотрит:
   - Сеть «замечает» → глитч-эффект (0.3s)
   - Переход в conservative режим:
     - Анимации замедляются
     - Движения становятся менее ровными
     - Цвета приглушаются
   - Detection Risk падает
4. Когда глаз отводит взгляд:
   - Сеть постепенно возвращается к нормальному режиму

### Технические детали

**Нейросеть:**
- Canvas или SVG с узлами и связями
- Узлы двигаются по случайным траекториям
- Связи пульсируют

**Глаз:**
- SVG-иконка глаза
- Анимация «взгляда» (зрачок двигается)
- При «взгляде на сеть» — зрачок направлен влево

**Глитч-эффект:**
- CSS filter + transform
- Кратковременное смещение RGB-каналов
- Дёрганье позиции

### Файлы
- `smart-committer.html`: разметка секции
- `smart-committer.css`: стили сети, глаза, глитча
- `smart-committer.js`: классы `NeuralNetworkViz`, `ObserverEye`, `AntidetectDemo`
